# CLAUDE.md — Sport_BB (адаптивная тренировочная программа)

Этот проект — персональная тренировочная система пользователя (не софт). Задача Claude: вести **адаптивную** программу — читать фидбек после тренировок и корректировать план с весами.

## Кто атлет
- Рост **183 см**, вес **85 кг**, жир **25–26%**. Тренируется «для себя», не проф. Русскоязычный — отвечать по-русски.
- **График:** через день (~3.5–4 сессии/нед), 1–1.5 ч.
- **Цели:** (1) полноценная масса / отличное телосложение; (2) прогресс в жиме лёжа, мечта — **жать 150 кг**.
- **Текущая сила (после 2 мес перерыва):** жим **~100 кг** с небольшим запасом (1ПМ ≈105–110); присед **60×8** с запасом (присед сильно детренирован относительно жима).
- **Предпочтения (обязательны в плане):** глубокий присед с подставкой под пятки, румынская тяга, горизонт. и вертик. тяга в тренажёре, жим на наклонной (верх груди отстаёт), махи на ширину плеч, бицепс, трицепс. Не перегружать: ~2 упр на большую мышцу, 1 на малую.

## Файлы
- `program/current-block.md` — **источник правды** по текущему блоку (у каждого упражнения прогрессия по неделям). Редактируем при корректировках.
- `program/current-block.html` — исходник интерактивного артефакта плана (редактируем и перевыпускаем вместе с .md).
- `log/training-log.md` — журнал тренировок и самочувствия; сюда заносим фидбек.
- `docs/cycle8-razbor.html` — исходник дашборда-разбора старого цикла + аудит + обоснование программы (опубликован как Artifact). **Не менять** без явной просьбы.
- `cycle8 ББ.xlsx` — старая программа (цикл Бондаренко), уже разобрана. Для чтения xlsx: venv с openpyxl (см. ниже).

## Модель планирования
Планируем **по одному мезоциклу за раз** (это удобно для адаптивности). Текущий = **Блок 1, 4 недели, схема 3+1** (нед.1–3 прогрессия, нед.4 разгрузка). Когда блок закончится → ретест жима/приседа → собрать Блок 2 от фактических цифр (можно архивировать старый в `program/`).

## Структура программы (Блок 1)
Верх/Низ, ротация `Верх A → Низ A → Верх B → Низ B`, через день. Каждая мышца 2×/нед.
- **Верх A:** жим лёжа тяж. 4×5 · наклонная 3×8 · гориз. тяга · верт. тяга · махи · трицепс-блок · бицепс-штанга.
- **Низ A:** глубокий присед 4×6 · РТ 3×8 · жим платформы · сгибание ног · икры стоя · пресс.
- **Верх B:** наклонная 4×8 (приоритет) · жим объём 3×8 · верт. тяга · гориз. тяга · махи · молотки · разгибания из-за головы.
- **Низ B:** РТ 4×8 (приоритет) · глубокий присед 3×8 · разгибание ног · сгибание ног · икры сидя · пресс.
Целевой объём/нед: грудь ~14, спина ~12, квадры ~13, задняя цепь ~13, плечи-ширина ~6, бицепс ~6, трицепс ~6 (+косвенно), икры ~6.

## Правила прогрессии (Блок 1)
- **Прогрессия по весу почти везде**, повторы фиксированы, +вес каждую неделю (нед.1→3), нед.4 разгрузка (вес назад к нед.1, 2 подхода). За неделю двигаем одну переменную.
  - Штанга/тренажёры/кабель: **+2.5 кг/нед** (жим платформы +10, икры +5).
  - **Гантельные изоляции** (боковые махи, молотки): вес фиксирован, **растут повторы** (шаг гантелей 2 кг груб для изоляции); вес +в Блоке 2.
  - Фолбэк: если минимальный скачок веса не даёт целевые повторы → +1–2 повтора вместо веса.
- Стартовые веса подсобки — оценочные (флаг «старт»), калибруются на нед.1 и заносятся в лог.
- **RIR:** нед.1 ≈3 · нед.2 ≈2 · нед.3 ≈1 · нед.4 ≈3–4. Без отказа в приседе/жиме со свободным весом без страховки.
- Точные веса всех упражнений по неделям — в `program/current-block.md`.

## Адаптивный цикл (главное!)
Пользователь будет открывать **новую сессию** и давать фидбек (что легко, что не смог, самочувствие, сон, боли). Тогда Claude:
1. Заносит фидбек в `log/training-log.md` (по шаблону оттуда).
2. Корректирует веса/объём в `program/12-week-plan.md` по правилам из лога (легко → быстрее вверх; не добрал/боль → откат/замена; плохое восстановление → снять объём / ранняя разгрузка).
3. Пишет строку в «Журнал изменений плана» в конце 12-week-plan.md.
4. При явной просьбе — перевыпускает Artifact плана (см. ниже).
Принцип: не ломать структуру без причины; менять цифры и упражнения точечно, объясняя почему. Не давать медицинских советов; при болях — осторожность и совет к специалисту.
