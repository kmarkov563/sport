# Задача B — подготовить/обновить будущие тренировки

Триггер: факт показал промах (сильно легко/тяжело, отказ, читинг, недоступный вес), атлет просит изменение, или следующий микроцикл не готов.

**Микроцикл и горизонт.** Один план = один микроцикл (сейчас — недельная ротация Верх A / Низ A / Верх B / Низ B, 4 тренировки). Всегда держать следующий микроцикл готовым целиком.

Прочитать: 
- `workout`/`notes` последних 4 микроциклов (если микроциклов меньше 4 — также `history/summary_<пол>.md`); 
- `requests/<пол>.txt` (цели, профиль, постоянные решения); 
- `base/common.txt` + `base/<пол>.txt` + `base/periodization.txt` + `base/exercises.txt` (другие файлы `base/` — `recovery` / `nutrition` / `sleep` — только если решение их касается).

1. Снять текущее состояние по каждому движению из прочитанных микроциклов (рабочий вес, RIR, шаги веса, снаряды).
2. Рассчитать нагрузку: цель блока и целевые параметры (объём на группу/нед, частота, диапазоны повторов, целевой RIR, отдых, модель прогрессии) из `requests/` и базы; индивидуализировать под стаж, детренированность, калибровку зала, шаги веса, боли, фидбек, постоянные решения. Крупный промах = заметная рекалибровка, а не шаг ±; заменённый снаряд калибровать как новый.
3. Записать микроцикл с явными числами и прогрессией вперёд:
   - нет микроцикла — составить сессии, каждой присвоить `globalNumber` из `nextGlobalNumber`, обновить `index.json` и `nextGlobalNumber`. `currentSessionId` должен указывать на ближайшую невыполненную сессию: если он остался на выполненной (микроцикл был закончен, задача A не смогла сдвинуть) — перевести на первую новую; если уже на невыполненной — не трогать;
   - есть — обновить только затронутые невыполненные `workout` (не сносить и строить заново). Изменённое значение в `workout` — и есть запись правки; отдельная заметка только если причина не видна ни в одном `notes`.

## Формат назначения

Только значимое для тренировки, кратко и чётко:

1. Упражнение и рабочие подходы построчно `вес × повторы × подходов, RIR N` (если RIR по подходам разный — свой для каждого). Подход, к которому есть пометка, отмечать `*`:
   - `85 кг × 5 × 1, RIR 1 *`
   - `80 кг × 5 × 3, RIR 2`
   - `* если 1-й подход легко — 2-й на 85`
2. Отдых между подходами.
3. Разминку атлет подбирает сам — отдельным разделом её не писать. Но если по модели нагрузки подводящие/разминочные подходы засчитываются в стимул, включай их в перечень рабочих подходов (пункт 1), а не как «разминку».

Атлет не думает — выполняет заданный вес и повторы. Никаких открытых диапазонов и «по самочувствию» («8–12 по самочувствию», «повышай постепенно»). Условие допустимо только конкретное, с числами («если легко — +2.5 кг»).

**Снаряд на выбор.** У каждого варианта свой вес и своя история прогрессии (45 в Смите ≠ 45 со свободной штангой). В JSON — отдельные блоки `loadBlocks` с `label`; калибровать и прогрессировать независимо, прогресс между снарядами не переносить.

Детали — `prescription-conventions.md`.

## Прогрессия и калибровка

Планировать по оптимистично-реалистичному сценарию нормального восстановления. Прогрессия не обязана быть одинаковой между упражнениями и тренировками.

**Смена зала.** Цифры на стеках разных тренажёров несопоставимы (конструкция, передача, трение, амплитуда). Исторический вес — грубый ориентир для первого назначения; дальше прогрессия строится от последнего факта на **этом** снаряде в **этом** зале. Меньшая/большая цифра после смены зала — не регресс/прогресс без учёта повторов, RIR, техники, амплитуды, фидбека. При замене тренажёра — рекалибровать заново.
